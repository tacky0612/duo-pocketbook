# 精算計算の仕様

このアプリケーションの中核となる計算。実装は `internal/domain/settlement.go` の `CalculateSettlement` にあり、**外部依存なしの純粋なドメインロジック**としてユニットテストで網羅している。

## 目的

ある月について、共有支出の立替額と収入をもとに、**精算後の可処分所得が指定した比重どおりになる**振込額を求める。

## 定義

| 記号 | 意味 |
|---|---|
| `incomeA`, `incomeB` | 各メンバーの対象月の収入 |
| `paidA`, `paidB` | 各メンバーが対象月に立て替えた共有支出の合計（固定費が登録されている場合、`SettlementUsecase` が対象月の支出として実体化して合算する） |
| `wA : wB` | 精算比重（正の整数）。**比重が大きい側が精算後に多く残る** |
| `netA = incomeA - paidA` | メンバーAの純額（精算前の可処分所得） |
| `t` | AからBへの振込額（負ならBからAへ振り込む） |

## 計算式

精算後の可処分所得を `dispA = netA - t`、`dispB = netB + t` とし、

```
dispA / wA == dispB / wB
```

を満たす `t` を解くと:

```
t = (wB × netA - wA × netB) / (wA + wB)
```

- **端数処理**: 1円未満は四捨五入（絶対値に対して half away from zero）。実装は `roundDiv`
- `t > 0` → A から B へ振込 / `t < 0` → B から A へ `|t|` 振込 / `t = 0` → 精算不要（`Transfer` は `nil`）

## 例

### 例1: 基本（比重1:1）— 要件の提示例

夫: 収入 100,000円・立替 20,000円 / 妻: 収入 50,000円・立替 20,000円

```
net夫 = 80,000 / net妻 = 30,000
t = (1×80,000 - 1×30,000) / 2 = 25,000
```

→ **夫が妻に25,000円振り込む**。双方の可処分所得は 55,000円 で等しくなる。

### 例2: 比重2:1

同じ収入・支出で比重を 夫2 : 妻1 にすると:

```
t = (1×80,000 - 2×30,000) / 3 = 6,666.67 → 6,667円（四捨五入）
```

→ 夫の可処分所得 73,333円、妻 36,667円（約2:1）。

### 例3: 逆方向

夫: 収入 50,000円・立替 30,000円 / 妻: 収入 100,000円・立替 0円（比重1:1）

```
t = (20,000 - 100,000) / 2 = -40,000
```

→ `t` が負なので**妻が夫に40,000円振り込む**。双方 60,000円になる。

## 性質と境界条件

- **保存則**: 精算は資金の移転のみであり、`dispA + dispB == netA + netB` が常に成り立つ（テストで検証）
- **支出ゼロでも精算は発生する**: 可処分所得を比重どおりに揃える仕様のため、収入差があれば支出がなくても振込額が算出される
- **収入0円は有効**（無収入の月もあり得る）。収入が負はバリデーションエラー
- **両メンバーの収入が未入力**の場合は `ErrIncomeNotReady` を返し、APIは `409 INCOME_NOT_READY` を返す
- 対象月以外の支出・収入が混入した場合、不明なメンバーが含まれる場合は `ErrValidation`

## 締め日

精算の対象期間はデフォルトで暦月（1日〜末日）だが、**締め日**（`/settings/closing-day`、`SETTINGS/CLOSINGDAY`）で起算日を変更できる。

- 締め日 D を設定すると、精算月 M の期間は「(M-1)月 D日 〜 M月 (D-1)日」になる。
  - 例: 締め日=15 → **7月の精算 = 6/15〜7/14** に記録した支出を計上する。
- 締め日=1（デフォルト）は暦月どおり（シフトなし）。
- 範囲は 1〜31。29〜31 はその日が存在しない月（2月など）では**その月の末日に丸める**（例: 締め日31・2月は28日/29日が起算）。
- 対象は支出のみ。収入・精算済みフラグ・固定費は月単位の設定なので締め日の影響を受けない（固定費は各精算月にそのまま計上）。
- 支出一覧・精算・履歴のいずれも同じ締め期間で集計する（画面間で金額・件数が一致する）。
- フロントの初期表示月も締め日に追従する。起動時に締め日を取得し、「今日が属する精算月」（`settlementMonthOf(今日, 締め日)`、`frontend/src/lib/month.ts`）を初期選択にする。例: 締め日=15・今日が7/20 なら初期表示は **8月**（7/15〜8/14 の期間）。締め日=1 なら暦当月と一致。ユーザーが月を手動変更した後や締め日の取得に失敗した場合は暦当月のままにする。

ドメインの `ClosingDay.SettlementMonth(date)`（`internal/domain/closing.go`）が支出日→精算月の対応を担い、`CalculateSettlement` は `SettlementInput.ClosingDay` で対象月に属するかを検証する。フロントの `settlementMonthOf`（`frontend/src/lib/month.ts`）はこれと同一ロジックの移植で、デモモードの支出絞り込みと初期表示月の算出に共用する。保存レイヤーでの扱い（暦月キーのまま2パーティションを集計）は [data-model.md](data-model.md#締め日は保存先を変えない暦月キーのまま集計時に期間で絞る) を参照。
